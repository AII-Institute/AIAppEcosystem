03 — Security Model
🪪
Identity & Access
Zero-trust · Single identity
OAuth 2.1 + PKCE — no legacy implicit flows
Single Sign-On across all ecosystem apps; one login, all surfaces
Short-lived JWTs (15 min) with rotating refresh tokens
MFA enforced at account level; passkey support
Per-app permission scopes — calendar can't read notes without consent
🔐
Transport Security
In-transit protection
TLS 1.3 mandatory on all connections; no downgrade allowed
Certificate pinning on mobile clients
MCP traffic signed with HMAC per request
E2E encryption for user messages (Signal Protocol)
HSTS headers enforced; DNSSEC on all domains
🧱
App Isolation
Least-privilege design
Each app runs as a separate MCP server with isolated credentials
Inter-app calls go through the MCP Router — never direct DB access
Row-level security in Postgres — users can only see their own data
AI agents operate in sandboxed tool scopes per session
Secrets in Vault (HashiCorp) — zero hardcoded credentials
🗝
Data Protection
At-rest encryption · Privacy
AES-256 encryption at rest; per-user key derivation (PBKDF2)
AI model never trained on user data without explicit opt-in
PII stripped from logs and telemetry pipelines automatically
Data residency options: user chooses region (EU / US / APAC)
Right-to-delete: full data purge within 30 days on request
🔎
Audit & Detection
Continuous monitoring
Immutable audit log for every MCP tool call and AI action
Anomaly detection on API access patterns (ML-based)
Prompt injection detection layer before LLM inference
Real-time alerts on unusual data volume or access velocity
Penetration testing quarterly; bug bounty program
📋
Compliance
Regulatory readiness
GDPR & CCPA: consent management, data portability built-in
EU Cyber Resilience Act compliance by design
SOC 2 Type II certification roadmap from day one
AI Act (EU) transparency requirements: users know when AI acts
Dependency scanning (Snyk/Dependabot) in CI/CD pipeline
Key Threats & Mitigations
Prompt Injection via MCP
HIGH
Malicious tool responses could hijack agent behavior.
Mitigation: Input sanitization layer + sandboxed tool execution + output validation before acting.
Token Exfiltration
HIGH
Stolen JWT could grant cross-app access.
Mitigation: Short-lived tokens, refresh rotation, device fingerprinting, anomaly detection on token use.
Data Leakage via RAG
HIGH
RAG could surface another user's data.
Mitigation: User-scoped embedding namespaces + row-level security enforced before retrieval.
Supply Chain Attack
MED
Compromised MCP server or npm package.
Mitigation: Signed packages, Sigstore, MCP registry verification, lockfile pinning.
Agent Scope Creep
MED
AI agent accessing unintended tools.
Mitigation: Per-session tool allowlists, human-in-the-loop for destructive actions, immutable audit log.
Offline Data Exposure
MED
Local on-device cache on lost/stolen device.
Mitigation: Encrypted local storage (Keychain/Keystore), remote wipe capability, biometric lock.
